# 🕵️‍♂️ Graph-RAG 기반 다자간 논리 추론 프레임워크
> **Multi-Agent Social Deduction & Logical Reasoning Framework**

본 프로젝트는 **AutoGen**과 **Graph-RAG(Neo4j + Milvus)**를 결합하여, 마피아 게임 시나리오 내에서 에이전트들이 논리적 모순을 탐지하고 "팀 가르기"를 통해 승리 조건을 달성하는 과정을 시뮬레이션합니다.

---

## 🎮 Game Flow & Rules

### 0. 논리적 상호작용 (Logical Interaction)
- 모든 에이전트는 말투, 감정 호소 등 게임과 무관한 비논리적 사고 및 발언을 금지합니다. (토큰 최적화 및 논리성 유지)

### 1. 승리 조건
- **마피아 팀**: 모든 시민의 직업을 맞추거나, 마피아의 수가 생존한 시민의 수와 같거나 많아질 경우.
- **시민 팀**: 모든 마피아 팀원을 색출하여 처단하거나 투표로 추방할 경우.

### 2. 역할 및 액션 (Roles & Actions)
- **경찰**: 매일 밤 무작위 에이전트 1인의 실제 직업 정보를 확인합니다.
- **마피아**: 매일 밤 제거할 에이전트 1인을 결정하며, 전략적으로 아군 정보를 공유합니다.
- **의무 공시**: 첫날 아침, 반드시 **진짜 경찰** 혹은 **사칭 경찰(마피아)** 중 최소 한 명은 자신의 신분을 공개하며 대화를 주도해야 합니다.

### 3. 게임 진행
- **추리**: 시민은 직업 공개를 최소화하며 마피아를 찾고, 마피아는 치명적인 가짜 정보를 유포합니다.
- **투표**: 낮 대화를 통해 과반수 표를 얻은 에이전트는 추방(사망) 처리됩니다.
- **순환**: 밤-낮-밤-낮 순으로 진행되며 승리 조건 충족 시 종료됩니다.

## 🧠 Logical Engine: "Team Partitioning"

단순 검색을 넘어, **Neo4j**의 그래프 구조를 활용하여 에이전트 간의 '팀' 관계를 수학적으로 도출합니다.

### 1. 대립 관계 형성 (Conflict Logic)
- **메커니즘**: 사칭 경찰이 진짜 경찰을 마피아로 지목하거나 동일 역할을 주장할 경우, 시스템은 두 에이전트 사이에 `[:CONFLICTS_WITH]` 관계를 형성합니다.
- **의미**: 이 관계가 형성된 두 에이전트는 절대 같은 팀일 수 없음을 의미합니다.

### 2. 사망 데이터를 통한 역추론 (Post-Mortem Inference)
- **메커니즘**: 밤사이 마피아에게 살해당한 에이전트는 시스템에 의해 **확정적 시민**으로 신분이 기록됩니다.
- **논리 확장**: 사망자가 시민으로 확정되는 즉시, 그와 대립(`:CONFLICTS_WITH`) 관계에 있던 생존 에이전트는 **확정적 마피아**로 즉시 식별됩니다.

// 에이전트 주장 및 대립 구조 모델링

(Agent)-[:CONFIRMED_AS]->(Status)   // 시스템 확정 신분
